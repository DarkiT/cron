# 任务注册功能

任务注册功能允许您预先定义任务，然后批量调度，简化任务管理，特别适合大型项目中的任务组织。

## 功能特性

- **预注册机制**: 在 init() 函数中注册任务，启动时批量调度
- **接口驱动**: 通过 RegisteredJob 接口统一任务定义
- **类型安全**: 编译时检查任务定义的完整性
- **批量调度**: 一次调用调度所有已注册任务
- **实例化注册表**: 推荐使用独立注册表，避免全局状态
- **向后兼容**: 保留全局注册用于简单场景

## 基本概念

### RegisteredJob 接口

RegisteredJob 是可注册任务的接口，继承自 Job 接口并添加调度信息：

```go
// Job 基础接口
type Job interface {
    Run(ctx context.Context) error  // 执行任务
    Name() string                   // 任务名称
}

// RegisteredJob 可注册任务接口
type RegisteredJob interface {
    Job
    Name() string     // 任务唯一标识
    Schedule() string // cron 调度表达式
}
```

### 任务注册表

```go
// JobRegistry 任务注册表
type JobRegistry struct {
    jobs map[string]RegisteredJob
    mu   sync.RWMutex
}

// 创建独立注册表
registry := cron.NewJobRegistry()
```

## 快速开始

### 推荐方式：实例化注册表

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

// 1. 定义任务
type BackupJob struct {
    path string
}

func (j *BackupJob) Name() string {
    return "backup-job"
}

func (j *BackupJob) Schedule() string {
    return "0 2 * * *"  // 每天 2 点执行
}

func (j *BackupJob) Run(ctx context.Context) error {
    fmt.Printf("执行备份: %s\n", j.path)
    return nil
}

// 2. 定义更多任务
type CleanupJob struct{}

func (j *CleanupJob) Name() string {
    return "cleanup-job"
}

func (j *CleanupJob) Schedule() string {
    return "0 3 * * *"  // 每天 3 点执行
}

func (j *CleanupJob) Run(ctx context.Context) error {
    fmt.Println("执行清理")
    return nil
}

func main() {
    // 3. 创建独立注册表（推荐）
    registry := cron.NewJobRegistry()

    // 4. 注册任务
    registry.SafeRegister(&BackupJob{path: "/data"})
    registry.SafeRegister(&CleanupJob{})

    // 5. 创建调度器
    c := cron.New()

    // 6. 批量调度注册表中的任务
    err := c.ScheduleFromRegistry(registry)
    if err != nil {
        fmt.Printf("调度失败: %v\n", err)
        return
    }

    // 7. 启动调度器
    c.Start()
    defer c.StopGracefully(5 * time.Second)

    // 保持运行
    select {}
}
```

### 兼容方式：全局注册

全局注册适合简单场景，但不推荐在复杂项目中使用：

```go
package jobs

import (
    "context"
    "fmt"

    "github.com/darkit/cron"

    return "@daily"
```

## 详细用法

### 实现 RegisteredJob

```go
// 示例 1: 简单任务
type SimpleJob struct {
    name string
}

func NewSimpleJob(name string) *SimpleJob {
    return &SimpleJob{name: name}
}

func (j *SimpleJob) Name() string {
    return j.name
}

func (j *SimpleJob) Schedule() string {
    return "*/5 * * * * *"  // 每 5 秒
}

func (j *SimpleJob) Run(ctx context.Context) error {
    fmt.Printf("任务 %s 执行\n", j.name)
    return nil
}

// 示例 2: 带配置的任务
type DatabaseSyncJob struct {
    db       *sql.DB
    interval string
}

func NewDatabaseSyncJob(db *sql.DB, interval string) *DatabaseSyncJob {
    return &DatabaseSyncJob{
        db:       db,
        interval: interval,
    }
}

func (j *DatabaseSyncJob) Name() string {
    return "db-sync"
}

func (j *DatabaseSyncJob) Schedule() string {
    return j.interval
}

func (j *DatabaseSyncJob) Run(ctx context.Context) error {
    // 同步数据库
    rows, err := j.db.QueryContext(ctx, "SELECT * FROM sync_table")
    if err != nil {
        return err
    }
    defer rows.Close()

    // 处理数据
    return nil
}

// 示例 3: 带依赖注入的任务
type EmailJob struct {
    mailer EmailSender
    logger Logger
}

func NewEmailJob(mailer EmailSender, logger Logger) *EmailJob {
    return &EmailJob{
        mailer: mailer,
        logger: logger,
    }
}

func (j *EmailJob) Name() string {
    return "email-sender"
}

func (j *EmailJob) Schedule() string {
    return "@hourly"
}

func (j *EmailJob) Run(ctx context.Context) error {
    j.logger.Info("开始发送邮件")

    if err := j.mailer.SendPendingEmails(ctx); err != nil {
        j.logger.Error("发送失败:", err)
        return err
    }

    j.logger.Info("发送完成")
    return nil
}
```

### 注册和调度

```go
func main() {
    // 初始化依赖
    db := initDatabase()
    mailer := initMailer()
    logger := initLogger()

    // 创建注册表
    registry := cron.NewJobRegistry()

    // 注册任务
    registry.SafeRegister(NewSimpleJob("job-1"))
    registry.SafeRegister(NewDatabaseSyncJob(db, "@every 10m"))
    registry.SafeRegister(NewEmailJob(mailer, logger))

    // 创建调度器
    c := cron.New()

    // 批量调度，可以为所有任务设置统一的默认选项
    err := c.ScheduleFromRegistry(registry, cron.JobOptions{
        Timeout: 5 * time.Minute,  // 统一超时
        Async:   true,             // 统一异步执行
    })
    if err != nil {
        log.Fatal(err)
    }

    // 启动
    c.Start()
    defer c.StopGracefully(10 * time.Second)

    select {}
}
```

### 条件注册

```go
func registerJobs(registry *cron.JobRegistry, config *Config) {
    // 始终注册核心任务
    registry.SafeRegister(NewCoreJob())

    // 根据配置注册可选任务
    if config.EnableBackup {
        registry.SafeRegister(NewBackupJob(config.BackupPath))
    }

    if config.EnableSync {
        registry.SafeRegister(NewSyncJob(config.SyncInterval))
    }

    if config.EnableCleanup {
        registry.SafeRegister(NewCleanupJob())
    }

    // 根据环境注册不同任务
    switch config.Environment {
    case "production":
        registry.SafeRegister(NewMonitorJob("prod"))
        registry.SafeRegister(NewAlertJob())
    case "staging":
        registry.SafeRegister(NewMonitorJob("staging"))
    case "development":
        registry.SafeRegister(NewDebugJob())
    }
}

func main() {
    config := loadConfig()
    registry := cron.NewJobRegistry()

    registerJobs(registry, config)

    c := cron.New()
    c.ScheduleFromRegistry(registry)
    c.Start()

    select {}
}
```

## 最佳实践

### 1. 工厂模式

使用工厂函数创建任务实例：

```go
// 定义工厂函数
func NewBackupJob(path string, interval string) *BackupJob {
    return &BackupJob{
        path:     path,
        interval: interval,
        logger:   log.Default(),
    }
}

// 使用工厂创建和注册
registry := cron.NewJobRegistry()
registry.SafeRegister(NewBackupJob("/data", "@daily"))
registry.SafeRegister(NewBackupJob("/logs", "@weekly"))
```

### 4. 任务生命周期

实现任务的初始化和清理：

```go
type ManagedJob struct {
    name   string
    client *http.Client
}

func NewManagedJob(name string) *ManagedJob {
    job := &ManagedJob{
        name: name,
    }
    job.Initialize()
    return job
}

func (j *ManagedJob) Initialize() {
    // 初始化资源
    j.client = &http.Client{
        Timeout: 30 * time.Second,
    }
}

func (j *ManagedJob) Cleanup() {
    // 清理资源
    j.client.CloseIdleConnections()
}

func (j *ManagedJob) Name() string {
    return j.name
}

func (j *ManagedJob) Schedule() string {
    return "@every 5m"
}

func (j *ManagedJob) Run(ctx context.Context) error {
    // 使用资源
    resp, err := j.client.Get("https://api.example.com/data")
    if err != nil {
        return err
    }
    defer resp.Body.Close()

    // 处理响应
    return nil
}

func main() {
    registry := cron.NewJobRegistry()
    job := NewManagedJob("api-poller")
    registry.SafeRegister(job)

    c := cron.New()
    c.ScheduleFromRegistry(registry)
    c.Start()

    // 优雅停止时清理资源
    defer func() {
        c.StopGracefully(10 * time.Second)
        job.Cleanup()
    }()

    select {}
}
```

### 5. 错误处理

在任务中实现健壮的错误处理：

```go
type ResilientJob struct {
    name    string
    logger  *log.Logger
    metrics *Metrics
}

func (j *ResilientJob) Run(ctx context.Context) error {
    start := time.Now()

    // 记录开始
    j.logger.Printf("任务 %s 开始执行", j.name)

    // 执行任务
    err := j.doWork(ctx)

    // 记录耗时
    duration := time.Since(start)
    j.metrics.RecordDuration(j.name, duration)

    // 错误处理
    if err != nil {
        j.logger.Printf("任务 %s 失败: %v", j.name, err)
        j.metrics.RecordFailure(j.name)
        return err
    }

    j.logger.Printf("任务 %s 成功完成，耗时 %v", j.name, duration)
    j.metrics.RecordSuccess(j.name)
    return nil
}

func (j *ResilientJob) doWork(ctx context.Context) error {
    // 检查上下文
    select {
    case <-ctx.Done():
        return ctx.Err()
    default:
    }

    // 实际工作
    // ...

    return nil
}
```

## 高级特性

### 动态任务注册

```go
// 从配置文件加载任务
func loadJobsFromConfig(configFile string, registry *cron.JobRegistry) error {
    data, err := os.ReadFile(configFile)
    if err != nil {
        return err
    }

    var config struct {
        Jobs []struct {
            Name     string `json:"name"`
            Type     string `json:"type"`
            Schedule string `json:"schedule"`
            Config   map[string]interface{} `json:"config"`
        } `json:"jobs"`
    }

    if err := json.Unmarshal(data, &config); err != nil {
        return err
    }

    for _, jobConfig := range config.Jobs {
        job, err := createJob(jobConfig.Type, jobConfig.Name, jobConfig.Schedule, jobConfig.Config)
        if err != nil {
            log.Printf("创建任务失败: %v", err)
            continue
        }

        registry.SafeRegister(job)
    }

    return nil
}

func createJob(jobType, name, schedule string, config map[string]interface{}) (cron.RegisteredJob, error) {
    switch jobType {
    case "backup":
        path := config["path"].(string)
        return NewBackupJob(name, schedule, path), nil
    case "cleanup":
        target := config["target"].(string)
        return NewCleanupJob(name, schedule, target), nil
    default:
        return nil, fmt.Errorf("unknown job type: %s", jobType)
    }
}
```

### 任务发现

```go
// 自动发现并注册实现了 RegisteredJob 接口的任务
func discoverAndRegisterJobs(registry *cron.JobRegistry, packages ...interface{}) {
    for _, pkg := range packages {
        v := reflect.ValueOf(pkg)
        t := v.Type()

        // 遍历包中的所有导出类型
        for i := 0; i < t.NumMethod(); i++ {
            method := t.Method(i)
            if method.Type.NumOut() == 1 {
                result := method.Func.Call([]reflect.Value{v})
                if job, ok := result[0].Interface().(cron.RegisteredJob); ok {
                    registry.SafeRegister(job)
                }
            }
        }
    }
}
```

## 实例化 vs 全局注册

### 对比

| 特性 | 实例化注册表 | 全局注册 |
|------|------------|---------|
| 推荐程度 | 强烈推荐 | 兼容性保留 |
| 测试友好 | ✅ 易于测试 | ❌ 测试困难 |
| 并发安全 | ✅ 完全隔离 | ⚠️ 需要注意 |
| 依赖注入 | ✅ 容易实现 | ❌ 不便实现 |
| 代码清晰 | ✅ 显式依赖 | ❌ 隐式依赖 |
| 适用场景 | 中大型项目 | 简单脚本 |

### 使用建议

**推荐：实例化注册表**

```go
registry := cron.NewJobRegistry()
registry.SafeRegister(job)
c.ScheduleFromRegistry(registry)
```

**不推荐：全局注册**（仅用于示例和简单场景）

```go
func init() {
    cron.SafeRegisterJob(job)
}
c.ScheduleRegistered()
```

## 常见问题

### Q: 如何选择注册方式？

A: 推荐使用实例化注册表（`NewJobRegistry()`），避免全局状态，便于测试和依赖管理。

### Q: 可以动态添加任务吗？

A: 可以。注册表只是批量调度的便利工具，您仍然可以使用 `Schedule` 方法动态添加任务。

### Q: 如何为不同任务设置不同的选项？

A: 批量调度时的 `JobOptions` 是默认选项。对于需要特殊配置的任务，可以单独调度：

```go
// 批量调度大部分任务
c.ScheduleFromRegistry(registry, cron.JobOptions{
    Timeout: 5 * time.Minute,
})

// 单独调度需要特殊配置的任务
c.ScheduleJob("special-task", "@every 1m", specialJob, cron.JobOptions{
    Timeout:       30 * time.Minute,
    MaxConcurrent: 1,
})
```

### Q: 任务注册失败会怎样？

A: 使用 `SafeRegister` 时，注册失败只会记录警告日志，不会 panic。使用 `register` 方法会返回错误。

### Q: 如何测试已注册的任务？

A: 使用实例化注册表便于测试：

```go
func TestJobExecution(t *testing.T) {
    registry := cron.NewJobRegistry()
    job := NewTestJob()
    registry.SafeRegister(job)

    c := cron.New()
    c.ScheduleFromRegistry(registry)

    // 测试任务执行
    err := job.Run(context.Background())
    assert.NoError(t, err)
}
```

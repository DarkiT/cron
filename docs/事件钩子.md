# 事件钩子

事件钩子（Event Hook）是一个强大的监控和扩展机制，允许你在任务执行的关键时刻注入自定义逻辑，实现日志记录、指标收集、告警通知等功能。

## 事件钩子系统介绍

### 什么是事件钩子

事件钩子是一个回调函数，在任务执行的特定时刻被调用，让你能够：

- 记录任务执行情况
- 收集性能指标
- 发送告警通知
- 与外部系统集成
- 实现自定义监控

### 事件触发时机

事件钩子在以下两个时刻被触发：

1. **任务开始时**：任务刚开始执行
2. **任务结束时**：任务执行完成（成功或失败）

## Event 类型结构说明

### Event 结构体

```go
type Event struct {
    TaskID   string        // 任务ID
    Start    time.Time     // 开始时间
    End      time.Time     // 结束时间（开始事件时为零值）
    Success  bool          // 是否成功
    Error    string        // 错误信息
    Retries  int           // 重试次数
    Duration time.Duration // 执行耗时（开始事件时为0）
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `TaskID` | string | 任务的唯一标识符 |
| `Start` | time.Time | 任务开始执行的时间 |
| `End` | time.Time | 任务结束的时间，开始事件时为零值 |
| `Success` | bool | 任务是否执行成功 |
| `Error` | string | 失败时的错误信息 |
| `Retries` | int | 总共重试了多少次 |
| `Duration` | time.Duration | 任务执行耗时（End - Start） |

### 区分开始和结束事件

```go
hook := func(ev cron.Event) {
    if ev.End.IsZero() {
        // 开始事件
        fmt.Printf("任务 %s 开始执行\n", ev.TaskID)
    } else {
        // 结束事件
        if ev.Success {
            fmt.Printf("任务 %s 成功完成，耗时 %v\n", ev.TaskID, ev.Duration)
        } else {
            fmt.Printf("任务 %s 执行失败: %s\n", ev.TaskID, ev.Error)
        }
    }
}
```

## WithEventHook 配置使用

### 基本用法

使用 `WithEventHook` 选项创建调度器时注册事件钩子：

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

func main() {
    // 定义事件钩子函数
    hook := func(ev cron.Event) {
        if ev.End.IsZero() {
            return // 忽略开始事件
        }

        if ev.Success {
            fmt.Printf("[成功] 任务=%s, 耗时=%v, 重试=%d\n",
                ev.TaskID, ev.Duration, ev.Retries)
        } else {
            fmt.Printf("[失败] 任务=%s, 耗时=%v, 重试=%d, 错误=%s\n",
                ev.TaskID, ev.Duration, ev.Retries, ev.Error)
        }
    }

    // 创建带事件钩子的调度器
    c := cron.New(cron.WithEventHook(hook))

    // 添加任务
    c.Schedule("task-1", "*/5 * * * * *", func(ctx context.Context) {
        fmt.Println("任务1执行")
    })

    c.Start()
    defer c.Stop()

    time.Sleep(30 * time.Second)
}
```

### 多个钩子组合

虽然 `WithEventHook` 只能设置一个钩子，但可以在钩子内部调用多个处理函数：

```go
func main() {
    // 组合多个处理器
    hook := func(ev cron.Event) {
        logEvent(ev)       // 记录日志
        collectMetrics(ev) // 收集指标
        sendAlert(ev)      // 发送告警
    }

    c := cron.New(cron.WithEventHook(hook))
    // ...
}

func logEvent(ev cron.Event) {
    // 日志处理逻辑
}

func collectMetrics(ev cron.Event) {
    // 指标收集逻辑
}

func sendAlert(ev cron.Event) {
    // 告警发送逻辑
}
```

## NewEventLoggerHook 使用示例

`NewEventLoggerHook` 是一个内置的事件钩子实现，将任务执行结果记录到日志。

### 基本使用

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

func main() {
    // 创建日志器
    logger := cron.NewDefaultLogger()

    // 创建日志钩子
    logHook := cron.NewEventLoggerHook(logger)

    // 使用日志钩子
    c := cron.New(
        cron.WithLogger(logger),
        cron.WithEventHook(logHook),
    )

    // 添加任务
    c.Schedule("demo-task", "*/10 * * * * *", func(ctx context.Context) {
        fmt.Println("演示任务执行")
        time.Sleep(2 * time.Second)
    })

    c.Start()
    defer c.Stop()

    time.Sleep(35 * time.Second)
}
```

### 输出示例

```
time=2025-10-30T18:00:00+08:00 level=INFO msg="task demo-task done, duration=2.001s, retries=0"
time=2025-10-30T18:00:10+08:00 level=INFO msg="task demo-task done, duration=2.002s, retries=0"
time=2025-10-30T18:00:20+08:00 level=INFO msg="task demo-task done, duration=2.001s, retries=0"
```

### 特点

- 仅记录结束事件，避免日志噪音
- 成功使用 `INFO` 级别
- 失败使用 `WARN` 级别
- 自动包含耗时和重试次数

## NewEventChannelHook 使用示例

`NewEventChannelHook` 将事件推送到 channel，方便集成到自己的监控系统。

### 基本使用

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

func main() {
    // 创建事件通道
    eventChan := make(chan cron.Event, 100)

    // 创建 channel 钩子
    channelHook := cron.NewEventChannelHook(eventChan)

    // 使用 channel 钩子
    c := cron.New(cron.WithEventHook(channelHook))

    // 启动事件处理器
    go processEvents(eventChan)

    // 添加任务
    c.Schedule("task-1", "*/5 * * * * *", func(ctx context.Context) {
        fmt.Println("任务1执行")
    })

    c.Schedule("task-2", "*/7 * * * * *", func(ctx context.Context) {
        fmt.Println("任务2执行")
        time.Sleep(1 * time.Second)
    })

    c.Start()
    defer c.Stop()

    time.Sleep(30 * time.Second)
}

func processEvents(eventChan <-chan cron.Event) {
    for ev := range eventChan {
        if ev.End.IsZero() {
            continue // 忽略开始事件
        }

        fmt.Printf("收到事件: 任务=%s, 成功=%t, 耗时=%v\n",
            ev.TaskID, ev.Success, ev.Duration)

        // 可以在这里：
        // - 写入数据库
        // - 发送到消息队列
        // - 更新监控系统
        // - 触发告警
    }
}
```

### 特点

- 非阻塞发送（超时10ms）
- 适合高频事件场景
- 易于集成外部系统

### 实时监控示例

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

type TaskMetrics struct {
    totalCount   int
    successCount int
    failCount    int
    totalDuration time.Duration
}

func main() {
    eventChan := make(chan cron.Event, 100)
    metrics := make(map[string]*TaskMetrics)

    // 启动统计协程
    go func() {
        ticker := time.NewTicker(10 * time.Second)
        defer ticker.Stop()

        for {
            select {
            case ev := <-eventChan:
                if ev.End.IsZero() {
                    continue
                }

                // 更新统计
                if metrics[ev.TaskID] == nil {
                    metrics[ev.TaskID] = &TaskMetrics{}
                }

                m := metrics[ev.TaskID]
                m.totalCount++
                m.totalDuration += ev.Duration

                if ev.Success {
                    m.successCount++
                } else {
                    m.failCount++
                }

            case <-ticker.C:
                // 打印统计信息
                fmt.Println("\n===== 任务执行统计 =====")
                for taskID, m := range metrics {
                    successRate := float64(m.successCount) / float64(m.totalCount) * 100
                    avgDuration := m.totalDuration / time.Duration(m.totalCount)

                    fmt.Printf("任务: %s\n", taskID)
                    fmt.Printf("  总次数: %d\n", m.totalCount)
                    fmt.Printf("  成功率: %.2f%%\n", successRate)
                    fmt.Printf("  平均耗时: %v\n", avgDuration)
                }
                fmt.Println("========================")
            }
        }
    }()

    c := cron.New(cron.WithEventHook(cron.NewEventChannelHook(eventChan)))

    c.Schedule("task-1", "*/3 * * * * *", func(ctx context.Context) {
        time.Sleep(500 * time.Millisecond)
    })

    c.Schedule("task-2", "*/5 * * * * *", func(ctx context.Context) {
        time.Sleep(1 * time.Second)
    })

    c.Start()
    defer c.Stop()

    time.Sleep(1 * time.Minute)
}
```

## 自定义钩子实现

### 示例1：告警钩子

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

// AlertHook 告警钩子
type AlertHook struct {
    threshold time.Duration // 慢查询阈值
    failAlert bool          // 是否告警失败任务
}

func NewAlertHook(threshold time.Duration, failAlert bool) cron.EventHook {
    h := &AlertHook{
        threshold: threshold,
        failAlert: failAlert,
    }
    return h.Handle
}

func (h *AlertHook) Handle(ev cron.Event) {
    if ev.End.IsZero() {
        return // 忽略开始事件
    }

    // 慢查询告警
    if ev.Duration > h.threshold {
        h.sendSlowAlert(ev)
    }

    // 失败告警
    if !ev.Success && h.failAlert {
        h.sendFailAlert(ev)
    }
}

func (h *AlertHook) sendSlowAlert(ev cron.Event) {
    fmt.Printf("[告警-慢任务] 任务=%s, 耗时=%v, 阈值=%v\n",
        ev.TaskID, ev.Duration, h.threshold)
    // 实际应用中：发送邮件、短信、钉钉等
}

func (h *AlertHook) sendFailAlert(ev cron.Event) {
    fmt.Printf("[告警-失败] 任务=%s, 错误=%s, 重试=%d\n",
        ev.TaskID, ev.Error, ev.Retries)
    // 实际应用中：发送邮件、短信、钉钉等
}

func main() {
    // 创建告警钩子：超过3秒告警，失败也告警
    alertHook := NewAlertHook(3*time.Second, true)

    c := cron.New(cron.WithEventHook(alertHook))

    // 添加慢任务
    c.Schedule("slow-task", "*/10 * * * * *", func(ctx context.Context) {
        time.Sleep(5 * time.Second) // 模拟慢任务
    })

    c.Start()
    defer c.Stop()

    time.Sleep(35 * time.Second)
}
```

### 示例3：多级钩子

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

// MultiHook 组合多个钩子
type MultiHook struct {
    hooks []cron.EventHook
}

func NewMultiHook(hooks ...cron.EventHook) cron.EventHook {
    return (&MultiHook{hooks: hooks}).Handle
}

func (m *MultiHook) Handle(ev cron.Event) {
    for _, hook := range m.hooks {
        // 每个钩子独立处理，避免一个失败影响其他
        func() {
            defer func() {
                if r := recover(); r != nil {
                    fmt.Printf("钩子执行panic: %v\n", r)
                }
            }()
            hook(ev)
        }()
    }
}

func main() {
    // 组合多个钩子
    logger := cron.NewDefaultLogger()
    eventChan := make(chan cron.Event, 100)

    multiHook := NewMultiHook(
        cron.NewEventLoggerHook(logger),
        cron.NewEventChannelHook(eventChan),
        NewAlertHook(3*time.Second, true),
    )

    c := cron.New(cron.WithEventHook(multiHook))

    c.Schedule("task-1", "*/5 * * * * *", func(ctx context.Context) {
        fmt.Println("任务执行")
    })

    c.Start()
    defer c.Stop()

    time.Sleep(20 * time.Second)
}

func NewAlertHook(threshold time.Duration, failAlert bool) cron.EventHook {
    return func(ev cron.Event) {
        if ev.End.IsZero() || ev.Duration <= threshold {
            return
        }
        fmt.Printf("[慢任务告警] %s 耗时 %v\n", ev.TaskID, ev.Duration)
    }
}
```

## 与 Prometheus/Grafana 集成示例

### Prometheus 指标收集

```go
package main

import (
    "context"
    "fmt"
    "net/http"
    "time"

    "github.com/darkit/cron"
    "github.com/prometheus/client_golang/prometheus"
    "github.com/prometheus/client_golang/prometheus/promhttp"
)

var (
    taskExecutionTotal = prometheus.NewCounterVec(
        prometheus.CounterOpts{
            Name: "cron_task_execution_total",
            Help: "Total number of task executions",
        },
        []string{"task_id", "status"},
    )

    taskExecutionDuration = prometheus.NewHistogramVec(
        prometheus.HistogramOpts{
            Name:    "cron_task_execution_duration_seconds",
            Help:    "Task execution duration in seconds",
            Buckets: prometheus.DefBuckets,
        },
        []string{"task_id"},
    )

    taskRetries = prometheus.NewHistogramVec(
        prometheus.HistogramOpts{
            Name:    "cron_task_retries",
            Help:    "Number of task retries",
            Buckets: []float64{0, 1, 2, 3, 5, 10},
        },
        []string{"task_id"},
    )
)

func init() {
    prometheus.MustRegister(taskExecutionTotal)
    prometheus.MustRegister(taskExecutionDuration)
    prometheus.MustRegister(taskRetries)
}

// PrometheusHook 创建 Prometheus 指标收集钩子
func PrometheusHook() cron.EventHook {
    return func(ev cron.Event) {
        if ev.End.IsZero() {
            return // 忽略开始事件
        }

        // 记录执行次数
        status := "success"
        if !ev.Success {
            status = "failure"
        }
        taskExecutionTotal.WithLabelValues(ev.TaskID, status).Inc()

        // 记录执行耗时
        taskExecutionDuration.WithLabelValues(ev.TaskID).Observe(ev.Duration.Seconds())

        // 记录重试次数
        taskRetries.WithLabelValues(ev.TaskID).Observe(float64(ev.Retries))
    }
}

func main() {
    // 启动 Prometheus HTTP 服务器
    go func() {
        http.Handle("/metrics", promhttp.Handler())
        fmt.Println("Prometheus metrics available at http://localhost:2112/metrics")
        http.ListenAndServe(":2112", nil)
    }()

    // 创建带 Prometheus 钩子的调度器
    c := cron.New(cron.WithEventHook(PrometheusHook()))

    // 添加任务
    c.Schedule("task-1", "*/5 * * * * *", func(ctx context.Context) {
        fmt.Println("任务1执行")
        time.Sleep(500 * time.Millisecond)
    })

    c.Schedule("task-2", "*/10 * * * * *", func(ctx context.Context) {
        fmt.Println("任务2执行")
        time.Sleep(2 * time.Second)
    }, cron.JobOptions{
        MaxRetries:    2,
        RetryInterval: 1 * time.Second,
    })

    c.Start()
    defer c.Stop()

    // 运行2分钟
    time.Sleep(2 * time.Minute)
}
```

type StructuredLogHook struct {

func NewStructuredLogHook(logger *log.Logger) cron.EventHook {

}

    structuredHook := NewStructuredLogHook(logger)

    c := cron.New(cron.WithEventHook(structuredHook))

        fmt.Println("任务执行")

## 最佳实践

### 1. 避免阻塞

事件钩子应该快速返回，耗时操作应该异步处理：

```go
// 不好的做法：阻塞钩子
hook := func(ev cron.Event) {
    sendEmailBlocking(ev) // 阻塞
}

// 好的做法：异步处理
hook := func(ev cron.Event) {
    go sendEmailAsync(ev) // 异步
}
```

### 2. 错误处理

钩子内部应该捕获 panic，避免影响任务执行：

```go
hook := func(ev cron.Event) {
    defer func() {
        if r := recover(); r != nil {
            log.Printf("事件钩子panic: %v", r)
        }
    }()

    // 钩子逻辑
    processEvent(ev)
}
```

### 3. 性能考虑

钩子会在每次任务执行时调用，注意性能影响：

```go
// 使用缓冲channel避免阻塞
eventChan := make(chan cron.Event, 1000)

// 使用对象池减少分配
var eventPool = sync.Pool{
    New: func() interface{} {
        return &Event{}
    },
}
```

### 4. 区分事件类型

根据需求决定是否处理开始事件：

```go
hook := func(ev cron.Event) {
    if ev.End.IsZero() {
        // 处理开始事件
        fmt.Printf("任务 %s 开始执行\n", ev.TaskID)
        return
    }

    // 处理结束事件
    fmt.Printf("任务 %s 执行完成\n", ev.TaskID)
}
```

### 5. 日志级别控制

根据事件类型选择合适的日志级别：

```go
hook := func(ev cron.Event) {
    if ev.End.IsZero() {
        return
    }

    if ev.Success {
        if ev.Duration > 10*time.Second {
            log.Warn("任务执行缓慢", "task", ev.TaskID, "duration", ev.Duration)
        } else {
            log.Debug("任务执行成功", "task", ev.TaskID)
        }
    } else {
        log.Error("任务执行失败", "task", ev.TaskID, "error", ev.Error)
    }
}
```

## 常见问题

### Q: 事件钩子会影响任务执行性能吗？

A: 会有轻微影响。建议在钩子中使用异步处理，避免阻塞任务执行。

### Q: 钩子中的错误会影响任务执行吗？

A: 不会。钩子是独立执行的，即使钩子失败，任务仍然正常完成。建议在钩子内部捕获错误。

### Q: 可以在钩子中修改任务状态吗？

A: 不可以。`Event` 是只读的，不应该在钩子中修改调度器状态。

### Q: NewEventLoggerHook 和 NewEventChannelHook 可以同时使用吗？

A: 不能直接同时使用，但可以通过自定义 MultiHook 组合多个钩子。

### Q: 钩子中可以调用调度器的方法吗？

A: 可以，但要注意避免死锁。建议使用异步方式调用，或仅调用查询类方法。

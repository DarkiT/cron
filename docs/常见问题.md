# Cron 调度库常见问题

本文档收集了 cron 调度库使用过程中的常见问题和解决方案，采用 Q&A 格式便于快速查找。

## 基础使用

### Q: 如何开始使用 cron 调度库？

**A:** 最简单的使用方式：

```go
package main

import (
    "context"
    "fmt"
    "time"
    "github.com/darkit/cron"
)

func main() {
    // 创建调度器
    c := cron.New()

    // 添加任务
    c.Schedule("hello", "*/5 * * * * *", func(ctx context.Context) {
        fmt.Println("每 5 秒执行一次")
    })

    // 启动调度器
    c.Start()
    defer c.StopGracefully(5 * time.Second)

    // 等待
    time.Sleep(30 * time.Second)
}
```

### Q: 支持哪些 cron 表达式格式？

**A:** 支持标准 5 段和 6 段格式，以及特殊描述符：

```go
// 6 段表达式（秒级）
"*/30 * * * * *"      // 每 30 秒
"0 */5 * * * *"       // 每 5 分钟
"0 0 8 * * *"         // 每天 8 点

// 5 段表达式（分钟级）
"*/5 * * * *"         // 每 5 分钟
"0 8 * * *"           // 每天 8 点

// 预定义描述符
"@hourly"             // 每小时
"@daily"              // 每天
"@every 5s"           // 每 5 秒

// 时区支持
"TZ=America/New_York 0 0 * * *"      // 纽约时间
"CRON_TZ=Asia/Shanghai 0 30 9 * * *" // 上海时间
```

### Q: 如何查看任务列表和下次执行时间？

**A:** 使用内置的管理方法：

```go
// 列出所有任务
tasks := c.List()
fmt.Printf("当前任务: %v\n", tasks)

// 获取下次执行时间
nextRun, err := c.NextRun("task-id")
if err == nil {
    fmt.Printf("下次执行: %v\n", nextRun)
}

// 获取任务统计
stats, ok := c.GetStats("task-id")
if ok {
    fmt.Printf("运行 %d 次, 成功 %d 次\n", stats.RunCount, stats.SuccessCount)
}
```

## 任务配置

### Q: 如何设置任务超时？

**A:** 使用 JobOptions 配置超时时间：

```go
c.Schedule("timeout-task", "@every 1m", func(ctx context.Context) {
    // 任务会在 30 秒后自动超时
    select {
    case <-time.After(2 * time.Minute):
        fmt.Println("任务完成")
    case <-ctx.Done():
        fmt.Println("任务超时")
    }
}, cron.JobOptions{
    Timeout: 30 * time.Second,
})
```

### Q: 如何实现任务失败重试？

**A:** 配置重试策略：

```go
// 固定次数重试
c.Schedule("retry-task", "@every 10s", func(ctx context.Context) {
    if err := doSomething(); err != nil {
        panic(err)  // 触发重试
    }
}, cron.JobOptions{
    MaxRetries:    3,               // 最多重试 3 次
    RetryInterval: 1 * time.Second, // 重试间隔 1 秒
})

// 无限重试（关键任务）
c.Schedule("critical", "@every 5m", handler, cron.JobOptions{
    MaxRetries:    -1,  // 无限重试
    RetryInterval: 5 * time.Second,
})

// 立即重试（不等待）
c.Schedule("fast", "@every 1m", handler, cron.JobOptions{
    MaxRetries:    5,
    RetryInterval: 0,  // 立即重试
})
```

### Q: 如何限制任务的并发执行？

**A:** 使用 MaxConcurrent 选项：

```go
// 最多同时运行 3 个实例
c.Schedule("limited-task", "*/5 * * * * *", func(ctx context.Context) {
    // 任务逻辑
}, cron.JobOptions{
    MaxConcurrent: 3,
})
```

### Q: 如何让耗时任务不阻塞调度器？

**A:** 使用异步执行：

```go
// 同步执行（默认）- 适合快速任务
c.Schedule("fast-task", "*/1 * * * * *", fastHandler)

// 异步执行 - 适合耗时任务
c.Schedule("slow-task", "*/5 * * * * *", slowHandler, cron.JobOptions{
    Async: true,  // 异步执行，不阻塞调度循环
})
```

## 安全和错误处理

### Q: 任务 panic 会导致程序崩溃吗？

**A:** 不会。所有任务都内置 panic 保护：

```go
// 即使任务 panic，程序也不会崩溃
c.Schedule("risky-task", "*/10 * * * * *", func(ctx context.Context) {
    if someCondition {
        panic("任务遇到问题")  // 不会让程序崩溃
    }
    fmt.Println("正常执行")
})
```

自定义 panic 处理：

```go
type MyPanicHandler struct{}

func (h *MyPanicHandler) HandlePanic(taskID string, panicValue interface{}, stack []byte) {
    log.Printf("任务 %s 异常: %v", taskID, panicValue)
    // 发送告警、记录日志等
}

c := cron.New(cron.WithPanicHandler(&MyPanicHandler{}))
```

### Q: 如何优雅地停止调度器？

**A:** 使用 StopGracefully 方法：

```go
// 创建调度器
c := cron.New()

// 添加任务并启动
c.Schedule("task", "*/5 * * * * *", handler)
c.Start()

// 优雅停止，等待正在执行的任务完成（最多等待 10 秒）
defer c.StopGracefully(10 * time.Second)
```

使用上下文管理生命周期：

```go
// 响应系统信号
ctx, cancel := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)
defer cancel()

// 调度器绑定到上下文
scheduler := cron.New(cron.WithContext(ctx))
scheduler.Start()

// 收到信号时自动停止
<-ctx.Done()
```

### Q: 如何处理任务执行超时？

**A:** 任务应该检查上下文取消信号：

```go
c.Schedule("timeout-aware", "@every 1m", func(ctx context.Context) {
    // 方式 1：使用 select 检查
    select {
    case <-time.After(30 * time.Second):
        fmt.Println("任务完成")
    case <-ctx.Done():
        fmt.Println("任务被取消或超时")
        return
    }

    // 方式 2：在循环中检查
    for i := 0; i < 1000; i++ {
        select {
        case <-ctx.Done():
            return  // 及时退出
        default:
            processItem(i)
        }
    }
}, cron.JobOptions{
    Timeout: 45 * time.Second,
})
```

## 运行时控制

### Q: 如何手动触发任务执行？

**A:** 使用 RunNow 方法：

```go
// 立即执行一次任务（不影响正常调度）
err := c.RunNow("task-id")
if err != nil {
    log.Printf("触发任务失败: %v", err)
}
```

### Q: 如何动态修改任务的调度时间？

**A:** 使用 Update 方法：

```go
// 修改调度表达式
err := c.Update("task-id", "*/10 * * * * *")
if err != nil {
    log.Printf("更新任务失败: %v", err)
}

// 同时更新配置
err = c.Update("task-id", "*/10 * * * * *", cron.JobOptions{
    Timeout: 60 * time.Second,
})
```

### Q: 如何暂停和恢复任务？

**A:** 使用 Pause 和 Resume 方法：

```go
// 暂停任务
err := c.Pause("task-id")
if err != nil {
    log.Printf("暂停任务失败: %v", err)
}

// 恢复任务
err = c.Resume("task-id")
if err != nil {
    log.Printf("恢复任务失败: %v", err)
}
```

### Q: 如何在运行时添加和删除任务？

**A:** 调度器支持动态管理：

```go
// 添加新任务
err := c.Schedule("new-task", "*/30 * * * * *", handler)

// 删除任务
err = c.Remove("task-id")
if err != nil {
    log.Printf("删除任务失败: %v", err)
}
```

## 高级功能

### Q: 如何启用历史记录功能？

**A:** 使用 HistoryRecorder：

```go
import (
    "github.com/darkit/cron"
    "github.com/darkit/cron/history"
)

// 创建存储
storage, err := history.NewFileStorage("./history")
if err != nil {
    log.Fatal(err)
}
defer storage.Close()

// 创建历史记录器
recorder := history.NewHistoryRecorder(storage)
defer recorder.Close()

// 创建启用历史记录的调度器
c := cron.New(cron.WithHistoryRecorder(recorder))
```

查询历史记录：

```go
// 查询最近 24 小时的记录
now := time.Now()
yesterday := now.Add(-24 * time.Hour)
records, err := c.QueryHistory(history.RecordFilter{
    StartTime: &yesterday,
    EndTime:   &now,
})

// 查询特定任务的失败记录
records, err = c.QueryHistory(history.RecordFilter{
    TaskID:     "task-id",
    FailedOnly: true,
})

// 统计记录数量
count, err := c.CountHistory(history.RecordFilter{
    TaskID: "task-id",
})
```

### Q: 如何使用任务注册功能？

**A:** 实现 RegisteredJob 接口：

```go
// 定义任务
type BackupJob struct {
    path string
}

func (j *BackupJob) Name() string {
    return "backup-job"
}

func (j *BackupJob) Schedule() string {
    return "0 2 * * *"  // 每天 2 点
}

func (j *BackupJob) Run(ctx context.Context) error {
    fmt.Printf("执行备份: %s\n", j.path)
    return nil
}

// 推荐方式：使用实例化注册表
func main() {
    // 创建独立注册表
    reg := cron.NewJobRegistry()
    reg.SafeRegister(&BackupJob{path: "/data"})

    // 批量调度
    c := cron.New()
    c.ScheduleFromRegistry(reg)
    c.Start()
}

// 兼容方式：使用全局注册
func init() {
    cron.SafeRegisterJob(&BackupJob{path: "/data"})
}

func main() {
    c := cron.New()
    c.ScheduleRegistered()
    c.Start()
}
```

### Q: 如何监控任务执行情况？

**A:** 使用事件钩子：

```go
// 定义事件处理函数
hook := func(ev cron.Event) {
    if ev.End.IsZero() {
        return  // 跳过开始事件
    }

    // 记录任务执行信息
    log.Printf("任务 %s: 成功=%v, 耗时=%v, 重试=%d",
        ev.TaskID, ev.Success, ev.Duration, ev.Retries)

    // 监控慢任务
    if ev.Duration > 30*time.Second {
        log.Printf("慢任务警告: %s", ev.TaskID)
    }

    // 监控失败任务
    if !ev.Success {
        log.Printf("任务失败: %s, 错误: %s", ev.TaskID, ev.Error)
    }
}

// 创建带事件钩子的调度器
c := cron.New(cron.WithEventHook(hook))

// 或使用内置的日志钩子
c := cron.New(cron.WithEventHook(cron.NewEventLoggerHook(logger)))
```

### Q: 如何处理 Misfire（错过的任务）？

**A:** 配置 Misfire 策略：

```go
// 跳过错过的任务（默认）
c.Schedule("task", "@every 10s", handler, cron.JobOptions{
    MisfirePolicy: cron.MisfireSkip,
})

// 补跑一次错过的任务
c.Schedule("task", "@every 10s", handler, cron.JobOptions{
    MisfirePolicy: cron.MisfireRunOnce,
})

// 尝试追赶所有错过的任务
c.Schedule("task", "@every 10s", handler, cron.JobOptions{
    MisfirePolicy: cron.MisfireCatchUp,
})
```

### Q: 如何实现失败熔断？

**A:** 配置失败阈值和暂停时长：

```go
c.Schedule("unstable-task", "@every 1m", handler, cron.JobOptions{
    FailThreshold: 5,                   // 连续失败 5 次
    FailWindow:    10 * time.Minute,    // 在 10 分钟窗口内
    PauseDuration: 30 * time.Minute,    // 自动暂停 30 分钟
})
```

## 常见错误

### Q: 为什么会出现 "task xxx already exists" 错误？

**A:** 任务 ID 重复。确保每个任务使用唯一的 ID：

```go
// ❌ 错误：ID 重复
c.Schedule("task", "*/5 * * * * *", handler1)
c.Schedule("task", "*/10 * * * * *", handler2)  // 错误：ID 已存在

// ✅ 正确：使用唯一 ID
c.Schedule("task1", "*/5 * * * * *", handler1)
c.Schedule("task2", "*/10 * * * * *", handler2)
```

### Q: 为什么会出现 "invalid cron spec" 错误？

**A:** cron 表达式格式错误。检查表达式格式：

```go
// ❌ 错误的格式
c.Schedule("task", "* * * *", handler)        // 缺少字段
c.Schedule("task", "*/5 * * * * * *", handler) // 字段过多

// ✅ 正确的格式
c.Schedule("task", "*/5 * * * *", handler)     // 5 段格式
c.Schedule("task", "*/5 * * * * *", handler)   // 6 段格式
c.Schedule("task", "@every 5s", handler)       // 描述符格式
```

### Q: 为什么任务没有按预期执行？

**A:** 检查以下几点：

```go
// 1. 确认调度器已启动
c.Start()

// 2. 检查任务是否被暂停
stats, ok := c.GetStats("task-id")
if ok && stats.Paused {
    c.Resume("task-id")  // 恢复任务
}

// 3. 检查下次执行时间
nextRun, err := c.NextRun("task-id")
if err == nil {
    fmt.Printf("下次执行: %v\n", nextRun)
}

// 4. 检查任务是否失败触发熔断
if stats.FailCount >= failThreshold {
    // 任务可能已被自动暂停
}
```

### Q: 为什么历史记录查询返回空？

**A:** 检查历史记录配置：

```go
// 1. 确认启用了历史记录
if c.recorder == nil {
    // 需要在创建调度器时启用历史记录
    storage, _ := history.NewFileStorage("./history")
    recorder := history.NewHistoryRecorder(storage)
    c = cron.New(cron.WithHistoryRecorder(recorder))
}

// 2. 检查查询条件
records, err := c.QueryHistory(history.RecordFilter{
    TaskID: "task-id",
    // 注意：StartTime 和 EndTime 可能过滤了所有记录
})

// 3. 确认任务已经执行过
stats, _ := c.GetStats("task-id")
fmt.Printf("任务已执行 %d 次\n", stats.RunCount)
```

## 性能优化

### Q: 如何提高调度器性能？

**A:** 参考以下优化建议：

```go
// 1. 避免过于频繁的调度
c.Schedule("task", "*/5 * * * * *", handler)  // 每 5 秒，而不是每秒

// 2. 耗时任务使用异步执行
c.Schedule("slow", "@every 1m", slowHandler, cron.JobOptions{
    Async: true,
})

// 3. 设置合理的超时
c.Schedule("task", "@every 1m", handler, cron.JobOptions{
    Timeout: 30 * time.Second,
})

// 4. 错开任务执行时间
c.Schedule("task1", "0 * * * * *", handler1)   // 0 秒
c.Schedule("task2", "20 * * * * *", handler2)  // 20 秒
c.Schedule("task3", "40 * * * * *", handler3)  // 40 秒

// 5. 定期清理历史记录
c.Schedule("cleanup", "@daily", func(ctx context.Context) {
    thirtyDaysAgo := time.Now().Add(-30 * 24 * time.Hour)
    c.CleanupHistory(thirtyDaysAgo)
})
```

### Q: 如何在 Docker 容器中使用？

**A:** 确保正确处理信号和优雅停止：

```go
func main() {
    // 响应 SIGTERM 信号
    ctx, cancel := signal.NotifyContext(
        context.Background(),
        os.Interrupt,
        syscall.SIGTERM,
    )
    defer cancel()

    // 创建调度器
    c := cron.New(cron.WithContext(ctx))

    // 添加任务
    c.Schedule("task", "@every 1m", handler)

    // 启动
    c.Start()

    // 等待信号
    <-ctx.Done()
    log.Println("收到停止信号，正在优雅关闭...")

    // 等待任务完成
    time.Sleep(5 * time.Second)
}
```

# 响应 SIGTERM

## 其他问题

### Q: 如何在测试中使用 cron？

**A:** 使用上下文控制测试时长：

```go
func TestCronTask(t *testing.T) {
    // 5 秒后自动停止
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()

    c := cron.New(cron.WithContext(ctx))

    var count atomic.Int64
    c.Schedule("test", "*/1 * * * * *", func(ctx context.Context) {
        count.Add(1)
    })

    c.Start()
    <-ctx.Done()

    // 验证执行次数
    assert.Equal(t, int64(5), count.Load())
}
```

### Q: 支持多实例部署吗？

**A:** cron 调度库本身是单机调度器，多实例部署需要外部协调：

- 使用分布式锁（Redis、etcd）确保任务只在一个实例执行
- 使用任务分片，不同实例执行不同任务
- 使用专门的分布式调度系统（如 XXL-Job）

简单的分布式锁示例：

```go
c.Schedule("distributed-task", "@every 1m", func(ctx context.Context) {
    // 获取分布式锁
    lock := acquireDistributedLock("task-lock")
    if lock == nil {
        return  // 其他实例正在执行
    }
    defer lock.Release()

    // 执行任务
    doWork()
})
```

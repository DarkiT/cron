# 时序图

## 📊 时序图说明

本文档展示了Cron调度库中关键业务流程的时序图，帮助理解各组件间的交互时序和数据流向。

## 🚀 任务创建和注册时序图

```mermaid
sequenceDiagram
    participant User as 用户应用
    participant API as API层
    participant Builder as 任务构建器
    participant Validator as 任务验证器
    participant Registry as 任务注册表
    participant Scheduler as 调度器
    participant Monitor as 监控系统
    
    User->>API: 创建任务请求
    Note over User,API: c.NewJob("task", "*/5 * * * * *")
    
    API->>Builder: 创建任务构建器
    Builder->>Builder: 设置任务配置
    Note over Builder: .Async().WithTimeout(30s)
    
    User->>Builder: 调用Do()方法
    Builder->>Validator: 验证任务配置
    
    alt 验证成功
        Validator->>Registry: 注册任务
        Registry->>Registry: 检查任务名重复
        
        alt 任务名唯一
            Registry->>Scheduler: 添加到调度队列
            Scheduler->>Monitor: 初始化任务统计
            Monitor->>API: 返回成功
            API->>User: 返回nil
        else 任务名重复
            Registry->>API: 返回ErrAlreadyRegister
            API->>User: 返回错误
        end
    else 验证失败
        Validator->>API: 返回验证错误
        API->>User: 返回错误
    end
```

## ⚡ 任务执行时序图

```mermaid
sequenceDiagram
    participant Timer as 定时器
    participant Scheduler as 调度器
    participant CondEngine as 条件引擎
    participant DepManager as 依赖管理器
    participant Executor as 执行器
    participant Monitor as 监控系统
    participant TaskFunc as 任务函数
    
    Timer->>Scheduler: 时间到达触发
    Scheduler->>Scheduler: 检查任务是否运行中
    
    alt 任务未运行
        Scheduler->>CondEngine: 检查执行条件
        CondEngine->>CondEngine: 评估所有条件
        
        alt 条件满足
            CondEngine->>DepManager: 检查任务依赖
            DepManager->>DepManager: 检查依赖任务状态
            
            alt 依赖满足
                DepManager->>Executor: 开始执行任务
                Executor->>Monitor: 记录开始执行
                
                par 异步执行
                    Executor->>TaskFunc: 调用任务函数
                    TaskFunc->>TaskFunc: 执行业务逻辑
                    
                    alt 执行成功
                        TaskFunc->>Executor: 返回成功
                        Executor->>Monitor: 记录执行成功
                    else 执行异常
                        TaskFunc->>Executor: 抛出异常
                        Executor->>Monitor: 记录执行失败
                    end
                and 超时监控
                    Executor->>Executor: 启动超时定时器
                    
                    alt 超时触发
                        Executor->>TaskFunc: 取消执行
                        Executor->>Monitor: 记录执行超时
                    end
                end
                
                Executor->>DepManager: 通知执行完成
                DepManager->>Scheduler: 通知依赖任务
                Scheduler->>Timer: 计算下次执行时间
            else 依赖不满足
                DepManager->>Monitor: 记录跳过执行
                DepManager->>Scheduler: 跳过本次执行
            end
        else 条件不满足
            CondEngine->>Monitor: 记录条件跳过
            CondEngine->>Scheduler: 跳过本次执行
        end
    else 任务正在运行
        Scheduler->>Monitor: 记录跳过执行
        Note over Scheduler,Monitor: 避免重复执行
    end
```

## 🔄 智能调度优化时序图

```mermaid
sequenceDiagram
    participant HealthMonitor as 健康监控
    participant StatsCollector as 统计收集器
    participant SmartScheduler as 智能调度器
    participant Optimizer as 优化器
    participant Scheduler as 调度器
    participant AlertManager as 告警管理
    
    loop 健康检查循环
        HealthMonitor->>StatsCollector: 收集系统指标
        StatsCollector->>StatsCollector: 计算成功率、响应时间等
        StatsCollector->>Health
        
        HealthMonitor->>HealthMonitor: 分析
        
        alt 系统健康
            HealthMonitor->>SmartScheduler: 报告健康状态
            Note over HealthMonitor,SmartScheduler: 状态: healthy
        else 发现性能问题
            HealthMonitor->>Optimizer: 触发优化流程
            
            alt 成功率过低
                Optimizer->>Optimizer: 分析失败原因
                Optimizer->>Scheduler: 调整超时设置
                Optimizer->>Scheduler: 优化重试策略
            else 并发过高
                Optimizer->>Scheduler: 降低并发限制
                Optimizer->>Scheduler: 启用负载均衡
            else 响应时间过长
                Optimizer->>Scheduler: 调整执行间隔
                Optimizer->>Scheduler: 启用优先级调度
            end
            
            Optimizer->>HealthMonitor: 应用优化配置
            HealthMonitor->>AlertManager: 发送优化通知
        else 系统异常
            HealthMonitor->>AlertManager: 发送告警
            AlertManager->>AlertManager: 处理告警逻辑
            Note over AlertManager: 邮件、Webhook等通知
        end
        
        HealthMonitor->>HealthMonitor: 等待下次检查
    end
```

## 📊 任务依赖执行时序图

```mermaid
sequenceDiagram
    participant TaskA as 任务A
    participant TaskB as 任务B (依赖A)
    participant TaskC as 任务C (依赖B)
    participant DepManager as 依赖管理器
    participant Executor as 执行器
    participant NotifyCenter as 通知中心
    
    Note over TaskA,TaskC: 任务依赖链: A -> B -> C
    
    par 任务A执行
        TaskA->>Executor: 开始执行
        Executor->>Executor: 执行任务A逻辑
        
        alt A执行成功
            Executor->>DepManager: 通知A执行成功
            DepManager->>NotifyCenter: 发布A完成事件
        else A执行失败
            Executor->>DepManager: 通知A执行失败
            DepManager->>NotifyCenter: 发布A失败事件
        end
    and 任务B等待
        TaskB->>DepManager: 检查依赖A状态
        DepManager->>TaskB: A未完成，等待
        
        NotifyCenter->>DepManager: 接收A完成事件
        
        alt A执行成功
            DepManager->>TaskB: 依赖满足，可以执行
            TaskB->>Executor: 开始执行任务B
            
            alt B执行成功
                Executor->>DepManager: 通知B执行成功
                DepManager->>NotifyCenter: 发布B完成事件
            else B执行失败
                Executor->>DepManager: 通知B执行失败
                DepManager->>NotifyCenter: 发布B失败事件
            end
        else A执行失败
            DepManager->>TaskB: 依赖失败，跳过执行
            Note over TaskB: 根据SkipOnDependencyFailure配置
        end
    and 任务C等待
        TaskC->>DepManager: 检查依赖B状态
        DepManager->>TaskC: B未完成，等待
        
        NotifyCenter->>DepManager: 接收B完成事件
        
        alt B执行成功
            DepManager->>TaskC: 依赖满足，可以执行
            TaskC->>Executor: 开始执行任务C
        else B执行失败或被跳过
            DepManager->>TaskC: 依赖失败，跳过执行
        end
    end
```

## � 条件执档行评估时序图

```mermaid
sequenceDiagram
    participant Scheduler as 调度器
    participant CondEngine as 条件引擎
    participant TimeCondition as 时间条件
    participant FileCondition as 文件条件
    participant CustomCondition as 自定义条件
    participant ContextProvider as 上下文提供者
    participant Executor as 执行器
    
    Scheduler->>CondEngine: 检查任务执行条件
    CondEngine->>ContextProvider: 获取执行上下文
    ContextProvider->>CondEngine: 返回当前时间、环境等
    
    par 并行评估所有条件
        CondEngine->>TimeCondition: 检查时间条件
        Note over TimeCondition: OnWeekdays(), OnBusinessHours()
        TimeCondition->>TimeCondition: 检查当前时间
        TimeCondition->>CondEngine: 返回条件结果
    and
        CondEngine->>FileCondition: 检查文件条件
        Note over FileCondition: OnFileExists("/path/to/file")
        FileCondition->>FileCondition: 检查文件是否存在
        FileCondition->>CondEngine: 返回条件结果
    and
        CondEngine->>CustomCondition: 检查自定义条件
        Note over CustomCondition: OnCustom(func() bool {...})
        CustomCondition->>CustomCondition: 执行自定义逻辑
        CustomCondition->>CondEngine: 返回条件结果
    end
    
    CondEngine->>CondEngine: 合并所有条件结果
    Note over CondEngine: 使用AND逻辑合并
    
    alt 所有条件满足
        CondEngine->>Scheduler: 返回true
        Scheduler->>Executor: 继续执行任务
    else 任一条件不满足
        CondEngine->>Scheduler: 返回false
        Scheduler->>Scheduler: 跳过本次执行
        Note over Scheduler: 记录跳过原因
    end
```

## 🔄 任务生命周期时序图

```mermaid
sequenceDiagram
    participant User as 用户
    participant API as API层
    participant Registry as 任务注册表
    participant Scheduler as 调度器
    participant Executor as 执行器
    participant Monitor as 监控系统
    
    %% 任务创建阶段
    User->>API: 创建任务
    API->>Registry: 注册任务
    Registry->>Scheduler: 加入调度队列
    Scheduler->>Monitor: 初始化统计
    Note over User,Monitor: 任务状态: Created
    
    %% 任务调度阶段
    Scheduler->>Scheduler: 等待执行时间
    Note over Scheduler: 任务状态: Scheduled
    
    %% 任务执行阶段
    Scheduler->>Executor: 触发执行
    Note over Executor: 任务状态: Running
    Executor->>Monitor: 更新运行状态
    
    alt 执行成功
        Executor->>Monitor: 记录成功
        Note over Monitor: 任务状态: Completed
    else 执行失败
        Executor->>Monitor: 记录失败
        Note over Monitor: 任务状态: Failed
    else 执行超时
        Executor->>Monitor: 记录超时
        Note over Monitor: 任务状态: Timeout
    end
    
    %% 任务清理阶段
    Monitor->>Scheduler: 通知执行完成
    Scheduler->>Scheduler: 计算下次执行
    Note over Scheduler: 任务状态: Scheduled (下次)
    
    %% 任务移除阶段
    User->>API: 移除任务
    API->>Registry: 注销任务
    Registry->>Scheduler: 从队列移除
    Scheduler->>Monitor: 清理统计
    Note over User,Monitor: 任务状态: Removed
```

## 📝 时序图说明

### 关键时序特点

1. **异步执行**: 任务执行采用异步模式，避免阻塞调度器
2. **并行处理**: 条件检查、依赖验证等支持并行处理
3. **事件驱动**: 基于事件通知机制实现组件间解耦
4. **超时控制**: 所有长时间操作都有超时保护机制

### 性能优化点

1. **批量操作**: 统计收集和存储更新采用批量模式
2. **缓存机制**: 频繁访问的数据使用缓存加速
3. **异步通知**: 事件通知采用异步方式避免阻塞
4. **资源池**: 使用对象池减少内存分配开销

### 错误处理策略

1. **优雅降级**: 组件故障时自动降级保证核心功能
2. **重试机制**: 临时故障自动重试恢复
3. **故障隔离**: 单个任务故障不影响其他任务
4. **监控告警**: 异常情况及时监控和告警

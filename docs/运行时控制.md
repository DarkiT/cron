# 运行时控制

运行时控制功能允许你在任务运行过程中动态地管理任务的执行状态，无需重启调度器。这为灵活的任务管理提供了强大的支持。

## 功能概述

运行时控制提供以下核心功能：

- `RunNow(id)` - 立即触发任务执行
- `Update(id, spec)` - 动态更新调度规则
- `Pause(id)` - 暂停任务
- `Resume(id)` - 恢复任务
- `PauseAll()` - 暂停所有任务
- `ResumeAll()` - 恢复所有任务
- `GetTask(id)` - 获取任务详情
- `GetAllTasks()` - 获取所有任务详情
- `StopGracefully(timeout)` - 优雅停止调度器

## 核心方法

### RunNow - 立即触发任务

立即执行指定任务一次，不影响其正常的调度计划。

**方法签名：**

```go
func (c *Cron) RunNow(id string) error
```

**参数：**
- `id` - 任务 ID

**返回值：**
- `error` - 如果任务不存在则返回错误

**使用示例：**

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

func main() {
    c := cron.New()

    // 添加一个每小时执行的任务
    c.Schedule("backup", "0 0 * * * *", func(ctx context.Context) {
        fmt.Println("执行备份任务")
    })

    c.Start()
    defer c.Stop()

    // 立即触发一次备份，不等待下一个小时
    if err := c.RunNow("backup"); err != nil {
        fmt.Printf("触发任务失败: %v\n", err)
    }

    time.Sleep(5 * time.Second)
}
```

**使用场景：**
- 手动触发定时任务
- 测试任务是否正常工作
- 响应用户请求立即执行某个操作
- 在系统初始化时立即执行首次同步

### Update - 动态更新调度规则

在运行时更新任务的 cron 表达式和配置选项，无需停止调度器。

**方法签名：**

```go
func (c *Cron) Update(id, schedule string, opts ...JobOptions) error
```

**参数：**
- `id` - 任务 ID
- `schedule` - 新的 cron 表达式
- `opts` - 可选的任务配置

**返回值：**
- `error` - 如果任务不存在或表达式无效则返回错误

**使用示例：**

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

func main() {
    c := cron.New()

    // 添加任务，初始每5分钟执行一次
    c.Schedule("sync", "*/5 * * * * *", func(ctx context.Context) {
        fmt.Println("执行同步任务")
    })

    c.Start()
    defer c.Stop()

    // 运行一段时间后
    time.Sleep(10 * time.Second)

    // 动态调整为每10秒执行一次
    if err := c.Update("sync", "*/10 * * * * *"); err != nil {
        fmt.Printf("更新任务失败: %v\n", err)
    }

    fmt.Println("任务调度已更新为每10秒执行一次")

    time.Sleep(30 * time.Second)
}
```

**更新配置选项示例：**

```go
// 更新调度表达式的同时修改超时时间
err := c.Update("heavy-task", "*/30 * * * * *", cron.JobOptions{
    Timeout:       60 * time.Second,
    MaxRetries:    5,
    RetryInterval: 5 * time.Second,
})
```

**使用场景：**
- 根据系统负载动态调整执行频率
- 在维护窗口期间降低任务频率
- 根据业务需求实时修改调度策略
- A/B 测试不同的调度配置

### Pause - 暂停任务

暂停任务的调度，任务将不再按计划执行，但不会被删除。

**方法签名：**

```go
func (c *Cron) Pause(id string) error
```

**参数：**
- `id` - 任务 ID

**返回值：**
- `error` - 如果任务不存在则返回错误

**使用示例：**

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

func main() {
    c := cron.New()

    c.Schedule("monitor", "*/5 * * * * *", func(ctx context.Context) {
        fmt.Println("监控任务执行")
    })

    c.Start()
    defer c.Stop()

    time.Sleep(12 * time.Second)

    // 暂停监控任务
    if err := c.Pause("monitor"); err != nil {
        fmt.Printf("暂停任务失败: %v\n", err)
    }

    fmt.Println("监控任务已暂停")

    // 暂停期间任务不会执行
    time.Sleep(20 * time.Second)

    // 恢复任务
    c.Resume("monitor")
    fmt.Println("监控任务已恢复")

    time.Sleep(20 * time.Second)
}
```

**使用场景：**
- 系统维护期间临时停止某些任务
- 响应资源告警，暂停消耗大的任务
- 手动控制任务的启停
- 调试时临时禁用某些任务

### Resume - 恢复任务

恢复之前暂停的任务，任务将继续按计划执行。

**方法签名：**

```go
func (c *Cron) Resume(id string) error
```

**参数：**
- `id` - 任务 ID

**返回值：**
- `error` - 如果任务不存在则返回错误

**使用示例：**

```go
// 恢复之前暂停的任务
if err := c.Resume("monitor"); err != nil {
    fmt.Printf("恢复任务失败: %v\n", err)
}

// 获取下次执行时间
next, _ := c.NextRun("monitor")
fmt.Printf("任务将在 %v 执行\n", next)
```

**[注意]：**
- Resume 会重新计算下次执行时间
- 暂停期间错过的执行不会补跑（除非配置了 MisfirePolicy）
- 任务恢复后立即生效

### PauseAll - 暂停所有任务

一键暂停所有任务的调度。

**方法签名：**

```go
func (c *Cron) PauseAll()
```

**使用示例：**

```go
// 进入维护模式，暂停所有任务
c.PauseAll()
fmt.Println("所有任务已暂停")

// 执行维护操作...

// 维护完成后恢复
c.ResumeAll()
```

**使用场景：**
- 系统维护期间一键暂停所有任务
- 紧急情况下快速停止所有定时任务
- 批量控制任务状态

### ResumeAll - 恢复所有任务

一键恢复所有已暂停的任务。

**方法签名：**

```go
func (c *Cron) ResumeAll()
```

**使用示例：**

```go
// 恢复所有任务
c.ResumeAll()
fmt.Println("所有任务已恢复")
```

### GetTask - 获取任务详情

获取指定任务的详细信息。

**方法签名：**

```go
func (c *Cron) GetTask(id string) (*TaskInfo, bool)
```

**参数：**
- `id` - 任务 ID

**返回值：**
- `*TaskInfo` - 任务详情
- `bool` - 任务是否存在

**TaskInfo 结构：**

```go
type TaskInfo struct {
    ID        string            // 任务ID
    Schedule  string            // cron表达式
    Options   JobOptions        // 任务配置
    Labels    map[string]string // 元数据标签
    NextRun   time.Time         // 下次执行时间
    IsPaused  bool              // 是否暂停
    IsRunning bool              // 是否正在运行
    CreatedAt time.Time         // 创建时间
}
```

**使用示例：**

```go
// 获取任务详情
if info, ok := c.GetTask("backup"); ok {
    fmt.Printf("任务ID: %s\n", info.ID)
    fmt.Printf("调度表达式: %s\n", info.Schedule)
    fmt.Printf("下次执行: %v\n", info.NextRun)
    fmt.Printf("是否暂停: %v\n", info.IsPaused)
    fmt.Printf("是否运行中: %v\n", info.IsRunning)
    fmt.Printf("创建时间: %v\n", info.CreatedAt)
} else {
    fmt.Println("任务不存在")
}
```

### GetAllTasks - 获取所有任务详情

获取所有任务的详细信息列表。

**方法签名：**

```go
func (c *Cron) GetAllTasks() []*TaskInfo
```

**返回值：**
- `[]*TaskInfo` - 所有任务的详情列表

**使用示例：**

```go
// 获取所有任务详情
tasks := c.GetAllTasks()
for _, info := range tasks {
    status := "运行中"
    if info.IsPaused {
        status = "已暂停"
    }
    fmt.Printf("任务 %s [%s]: 下次执行 %v\n", info.ID, status, info.NextRun)
}

// 统计暂停的任务数量
pausedCount := 0
for _, info := range tasks {
    if info.IsPaused {
        pausedCount++
    }
}
fmt.Printf("共 %d 个任务，其中 %d 个已暂停\n", len(tasks), pausedCount)
```

### StopGracefully - 优雅停止

带超时的优雅停止调度器，等待正在执行的任务完成后再停止。

**方法签名：**

```go
func (c *Cron) StopGracefully(timeout time.Duration)
```

**参数：**
- `timeout` - 最大等待时间，0 或负值表示无限等待

**使用示例：**

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

func main() {
    c := cron.New()

    // 添加一个耗时任务
    c.Schedule("long-task", "*/5 * * * * *", func(ctx context.Context) {
        fmt.Println("开始执行耗时任务...")
        time.Sleep(3 * time.Second)
        fmt.Println("耗时任务完成")
    }, cron.JobOptions{
        Async: true, // 异步执行
    })

    c.Start()

    // 运行一段时间
    time.Sleep(12 * time.Second)

    // 优雅停止，等待最多10秒让任务完成
    fmt.Println("开始优雅停止调度器...")
    c.StopGracefully(10 * time.Second)
    fmt.Println("调度器已停止，所有任务已完成或超时")
}
```

**超时处理：**

```go
// 等待最多30秒
c.StopGracefully(30 * time.Second)

// 无限等待直到所有任务完成
c.StopGracefully(0)

// 立即停止（不等待）
c.Stop()
```

**使用场景：**
- 应用程序关闭时确保任务完成
- 滚动更新时的优雅下线
- 避免任务被强制中断造成数据不一致
- 响应系统信号（如 SIGTERM）时的优雅退出

## 完整示例

### 示例1：Web API 控制任务

```go
package main

import (
    "context"
    "encoding/json"
    "fmt"
    "net/http"
    "time"

    "github.com/darkit/cron"
)

var scheduler *cron.Cron

func main() {
    scheduler = cron.New()

    // 添加一些任务
    scheduler.Schedule("task-1", "*/10 * * * * *", func(ctx context.Context) {
        fmt.Println("任务1执行")
    })

    scheduler.Schedule("task-2", "*/20 * * * * *", func(ctx context.Context) {
        fmt.Println("任务2执行")
    })

    scheduler.Start()
    defer scheduler.StopGracefully(5 * time.Second)

    // 设置 HTTP API
    http.HandleFunc("/tasks/run", handleRunNow)
    http.HandleFunc("/tasks/pause", handlePause)
    http.HandleFunc("/tasks/resume", handleResume)
    http.HandleFunc("/tasks/update", handleUpdate)

    fmt.Println("API 服务启动在 :8080")
    http.ListenAndServe(":8080", nil)
}

func handleRunNow(w http.ResponseWriter, r *http.Request) {
    taskID := r.URL.Query().Get("id")
    if taskID == "" {
        http.Error(w, "缺少任务ID", http.StatusBadRequest)
        return
    }

    if err := scheduler.RunNow(taskID); err != nil {
        http.Error(w, err.Error(), http.StatusNotFound)
        return
    }

    json.NewEncoder(w).Encode(map[string]string{
        "message": "任务已触发",
        "task_id": taskID,
    })
}

func handlePause(w http.ResponseWriter, r *http.Request) {
    taskID := r.URL.Query().Get("id")
    if taskID == "" {
        http.Error(w, "缺少任务ID", http.StatusBadRequest)
        return
    }

    if err := scheduler.Pause(taskID); err != nil {
        http.Error(w, err.Error(), http.StatusNotFound)
        return
    }

    json.NewEncoder(w).Encode(map[string]string{
        "message": "任务已暂停",
        "task_id": taskID,
    })
}

func handleResume(w http.ResponseWriter, r *http.Request) {
    taskID := r.URL.Query().Get("id")
    if taskID == "" {
        http.Error(w, "缺少任务ID", http.StatusBadRequest)
        return
    }

    if err := scheduler.Resume(taskID); err != nil {
        http.Error(w, err.Error(), http.StatusNotFound)
        return
    }

    json.NewEncoder(w).Encode(map[string]string{
        "message": "任务已恢复",
        "task_id": taskID,
    })
}

func handleUpdate(w http.ResponseWriter, r *http.Request) {
    if r.Method != "POST" {
        http.Error(w, "仅支持POST请求", http.StatusMethodNotAllowed)
        return
    }

    var req struct {
        TaskID   string `json:"task_id"`
        Schedule string `json:"schedule"`
    }

    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        http.Error(w, "无效的请求体", http.StatusBadRequest)
        return
    }

    if err := scheduler.Update(req.TaskID, req.Schedule); err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)
        return
    }

    json.NewEncoder(w).Encode(map[string]string{
        "message":  "任务已更新",
        "task_id":  req.TaskID,
        "schedule": req.Schedule,
    })
}
```

**API 调用示例：**

```bash
# 立即触发任务

# 暂停任务
```

### 示例2：自适应调度

根据系统负载动态调整任务执行频率：

```go
package main

import (
    "context"
    "fmt"
    "math/rand"
    "time"

    "github.com/darkit/cron"
)

func main() {
    c := cron.New()

    // 添加数据同步任务
    c.Schedule("data-sync", "*/30 * * * * *", func(ctx context.Context) {
        fmt.Println("执行数据同步")
        time.Sleep(2 * time.Second)
    })

    c.Start()
    defer c.StopGracefully(5 * time.Second)

    // 监控系统负载并动态调整
    go func() {
        ticker := time.NewTicker(1 * time.Minute)
        defer ticker.Stop()

        for range ticker.C {
            // 模拟获取系统负载
            load := rand.Float64()

            var newSchedule string
            switch {
            case load > 0.8:
                // 高负载：降低频率到每2分钟
                newSchedule = "0 */2 * * * *"
                fmt.Println("系统负载高，降低同步频率")
            case load > 0.5:
                // 中等负载：每分钟执行
                newSchedule = "0 * * * * *"
                fmt.Println("系统负载中等，使用正常频率")
            default:
                // 低负载：每30秒执行
                newSchedule = "*/30 * * * * *"
                fmt.Println("系统负载低，提高同步频率")
            }

            if err := c.Update("data-sync", newSchedule); err != nil {
                fmt.Printf("更新调度失败: %v\n", err)
            }
        }
    }()

    // 运行5分钟
    time.Sleep(5 * time.Minute)
}
```

### 示例3：维护窗口控制

在维护窗口期间自动暂停和恢复任务：

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/darkit/cron"
)

func main() {
    c := cron.New()

    // 添加多个任务
    tasks := []string{"backup", "cleanup", "report"}
    for _, taskID := range tasks {
        id := taskID
        c.Schedule(id, "*/5 * * * * *", func(ctx context.Context) {
            fmt.Printf("执行任务: %s\n", id)
        })
    }

    c.Start()
    defer c.StopGracefully(5 * time.Second)

    // 定义维护窗口：每天凌晨2点到4点
    go func() {
        for {
            now := time.Now()

            // 检查是否在维护窗口内
            if now.Hour() >= 2 && now.Hour() < 4 {
                fmt.Println("进入维护窗口，暂停所有任务")
                for _, taskID := range tasks {
                    c.Pause(taskID)
                }

                // 等到维护窗口结束
                endTime := time.Date(now.Year(), now.Month(), now.Day(), 4, 0, 0, 0, now.Location())
                time.Sleep(time.Until(endTime))

                fmt.Println("维护窗口结束，恢复所有任务")
                for _, taskID := range tasks {
                    c.Resume(taskID)
                }
            }

            // 每小时检查一次
            time.Sleep(1 * time.Hour)
        }
    }()

    // 持续运行
    select {}
}
```

### 5. 优雅停止

```go
import (
    "os"
    "os/signal"
    "syscall"
)

func main() {
    c := cron.New()
    // ... 添加任务 ...
    c.Start()

    // 等待中断信号

    signal.Notify(sigChan, syscall.SIGINT, syscall.SIGTERM)

}
```

## 常见问题

### Q: RunNow 会影响任务的正常调度吗？

A: 不会。`RunNow` 是一次性的手动触发，不会改变任务的下次执行时间。任务仍然会按原定计划继续执行。

### Q: Update 更新后任务会立即执行吗？

A: 不会。`Update` 会重新计算下次执行时间，任务会在新的调度计划时间执行。如果需要立即执行，可以在更新后调用 `RunNow`。

### Q: Pause 后任务能否接收到 RunNow 触发？

A: 可以。`Pause` 只影响自动调度，手动调用 `RunNow` 仍然有效。

### Q: StopGracefully 超时后会发生什么？

A: 超时后，调度器会强制停止，正在执行的任务将收到上下文取消信号。任务应该监听 `ctx.Done()` 来优雅处理取消。

### Q: 可以在任务执行过程中调用 Update 吗？

A: 可以。`Update` 是线程安全的，可以在任何时候调用。正在执行的任务不会受影响，新的调度规则从下次执行开始生效。

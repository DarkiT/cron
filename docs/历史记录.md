# 历史记录功能

历史记录功能提供任务执行历史的持久化存储、查询和统计能力，帮助您追踪任务执行情况、分析问题和审计操作。

## 功能特性

- **持久化存储**: 基于 JSONL 格式的文件存储，可靠且易于维护
- **按需查询**: 支持按任务、时间范围、成功/失败状态等多维度查询
- **高效分片**: 按任务和日期分片存储，提升查询性能
- **异步写入**: 使用异步队列，不阻塞任务执行
- **自动清理**: 支持定期清理旧记录，控制存储空间
- **零依赖**: 基于标准库实现，无需外部数据库

## 快速开始

### 基本使用

```go
package main

import (
    "context"
    "fmt"
    "log"
    "time"

    "github.com/darkit/cron"
    "github.com/darkit/cron/history"
)

func main() {
    // 1. 创建历史记录存储
    storage, err := history.NewFileStorage("./history")
    if err != nil {
        log.Fatal(err)
    }
    defer storage.Close()

    // 2. 创建历史记录器
    recorder := history.NewHistoryRecorder(storage)
    defer recorder.Close()

    // 3. 创建启用历史记录的调度器
    c := cron.New(cron.WithHistoryRecorder(recorder))

    // 4. 添加任务
    c.Schedule("demo-task", "@every 5s", func(ctx context.Context) {
        fmt.Println("任务执行")
    })

    // 5. 启动调度器
    c.Start()
    defer c.StopGracefully(5 * time.Second)

    // 等待一段时间后查询历史
    time.Sleep(30 * time.Second)

    // 6. 查询历史记录
    records, err := c.QueryHistory(history.RecordFilter{
        TaskID: "demo-task",
        Limit:  10,
    })
    if err != nil {
        log.Printf("查询失败: %v", err)
        return
    }

    // 7. 显示结果
    fmt.Printf("找到 %d 条记录:\n", len(records))
    for _, record := range records {
        fmt.Printf("  时间: %v, 成功: %v, 耗时: %v\n",
            record.StartTime.Format("15:04:05"),
            record.Success,
            record.Duration,
        )
    }
}
```

## 存储格式

### JSONL 存储结构

历史记录采用 JSONL (JSON Lines) 格式存储，每行一条 JSON 记录：

```
<baseDir>/
  ├── task-id-1/
  │   ├── 2025-01-01.jsonl
  │   ├── 2025-01-02.jsonl
  │   └── 2025-01-03.jsonl
  ├── task-id-2/
  │   ├── 2025-01-01.jsonl
  │   └── 2025-01-02.jsonl
  └── ...
```

### 记录格式

每条记录包含以下字段：

```go
type ExecutionRecord struct {
    ID         string        // 记录唯一标识 (UUID)
    TaskID     string        // 任务 ID
    StartTime  time.Time     // 开始时间
    EndTime    time.Time     // 结束时间
    Duration   time.Duration // 执行耗时
    Success    bool          // 是否成功
    RetryCount int           // 重试次数
    Error      string        // 错误信息（如果失败）
}
```


### 存储优势

- **追加写入**: 只需追加新行，无需重写整个文件
- **易于查看**: 可直接使用 `cat`、`tail`、`grep` 等命令查看
- **容错性好**: 单条记录损坏不影响其他记录
- **高效查询**: 按日期分片，快速定位目标文件

### 使用清理脚本

项目提供了命令行清理脚本：

```bash
# 清理 30 天前的历史记录
./scripts/cleanup_history.sh ./history 30

# 清理 7 天前的历史记录
./scripts/cleanup_history.sh ./history 7

# 查看帮助
./scripts/cleanup_history.sh --help
```

清理脚本内容（`scripts/cleanup_history.sh`）：

```bash
#!/bin/bash
# 历史记录清理脚本

if [ "$#" -lt 2 ]; then
    echo "用法: $0 <history_dir> <keep_days>"
    echo "示例: $0 ./history 30"
    exit 1
fi

HISTORY_DIR="$1"
KEEP_DAYS="$2"

if [ ! -d "$HISTORY_DIR" ]; then
    echo "错误: 目录 $HISTORY_DIR 不存在"
    exit 1
fi

# 计算截止日期
CUTOFF_DATE=$(date -d "$KEEP_DAYS days ago" +%Y-%m-%d)

echo "清理 $CUTOFF_DATE 之前的历史记录..."

# 遍历所有任务目录
for task_dir in "$HISTORY_DIR"/*; do
    if [ -d "$task_dir" ]; then
        task_name=$(basename "$task_dir")
        echo "处理任务: $task_name"

        # 查找并删除旧文件
        find "$task_dir" -type f -name "*.jsonl" | while read file; do
            file_date=$(basename "$file" .jsonl)
            if [[ "$file_date" < "$CUTOFF_DATE" ]]; then
                echo "  删除: $file"
                rm -f "$file"
            fi
        done

        # 如果目录为空，删除目录
        if [ -z "$(ls -A "$task_dir")" ]; then
            echo "  删除空目录: $task_dir"
            rmdir "$task_dir"
        fi
    fi
done

echo "清理完成"
```

### 存储空间管理

```go
// 监控存储空间使用情况
func monitorStorage(historyDir string) {
    ticker := time.NewTicker(time.Hour)
    defer ticker.Stop()

    for range ticker.C {
        size, err := getDirSize(historyDir)
        if err != nil {
            log.Printf("获取目录大小失败: %v", err)
            continue
        }

        // 转换为 MB
        sizeMB := float64(size) / 1024 / 1024
        log.Printf("历史记录存储: %.2f MB", sizeMB)

        // 如果超过 1GB，触发清理
        if sizeMB > 1024 {
            log.Println("存储空间超过 1GB，开始清理...")
            // 清理 60 天前的记录
            sixtyDaysAgo := time.Now().Add(-60 * 24 * time.Hour)
            deleted, _ := c.CleanupHistory(sixtyDaysAgo)
            log.Printf("已清理 %d 条记录", deleted)
        }
    }
}

func getDirSize(path string) (int64, error) {
    var size int64
    err := filepath.Walk(path, func(_ string, info os.FileInfo, err error) error {
        if err != nil {
            return err
        }
        if !info.IsDir() {
            size += info.Size()
        }
        return nil
    })
    return size, err
}
```

```go
// 生成任务执行报告
func generateReport(c *cron.Cron, taskID string) {
    // 查询最近 7 天的记录
    sevenDaysAgo := time.Now().Add(-7 * 24 * time.Hour)
    now := time.Now()

    records, err := c.QueryHistory(history.RecordFilter{
        TaskID:    taskID,
        StartTime: &sevenDaysAgo,
        EndTime:   &now,
    })
    if err != nil {
        log.Printf("查询失败: %v", err)
        return
    }

    // 统计数据
    var (
        total       = len(records)
        successCount = 0
        failedCount = 0
        totalDuration time.Duration
        maxDuration time.Duration
        minDuration = time.Hour * 24
        totalRetries = 0
    )

    for _, record := range records {
        if record.Success {
            successCount++
        } else {
            failedCount++
        }

        totalDuration += record.Duration
        if record.Duration > maxDuration {
            maxDuration = record.Duration
        }
        if record.Duration < minDuration {
            minDuration = record.Duration
        }

        totalRetries += record.RetryCount
    }

    // 输出报告
    fmt.Printf("=== 任务执行报告: %s ===\n", taskID)
    fmt.Printf("统计时间: %v ~ %v\n", sevenDaysAgo.Format("2006-01-02"), now.Format("2006-01-02"))
    fmt.Printf("总执行次数: %d\n", total)
    fmt.Printf("成功次数: %d\n", successCount)
    fmt.Printf("失败次数: %d\n", failedCount)
    fmt.Printf("成功率: %.2f%%\n", float64(successCount)/float64(total)*100)
    fmt.Printf("平均耗时: %v\n", totalDuration/time.Duration(total))
    fmt.Printf("最长耗时: %v\n", maxDuration)
    fmt.Printf("最短耗时: %v\n", minDuration)
    fmt.Printf("总重试次数: %d\n", totalRetries)
    fmt.Printf("平均重试次数: %.2f\n", float64(totalRetries)/float64(total))
}
```

## 性能考虑

### 异步写入

历史记录器使用异步队列写入，不会阻塞任务执行：

```go
// 内部实现（简化）
type HistoryRecorder struct {
    queue   chan *ExecutionRecord
    storage Storage
}

func (r *HistoryRecorder) Record(record *ExecutionRecord) {
    select {
    case r.queue <- record:
        // 成功入队
    default:
        // 队列满，丢弃记录（可配置策略）
        log.Println("历史记录队列已满，丢弃记录")
    }
}

func (r *HistoryRecorder) worker() {
    for record := range r.queue {
        if err := r.storage.Save(record); err != nil {
            log.Printf("保存历史记录失败: %v", err)
        }
    }
}
```

## 常见问题

### Q: 历史记录会影响性能吗？

A: 不会。历史记录使用异步队列写入，不阻塞任务执行。即使写入失败，也不会影响任务的正常调度。

### Q: 如何限制历史记录的存储空间？

A: 使用定期清理任务：

```go
c.Schedule("cleanup", "@daily", func(ctx context.Context) {
    // 保留最近 30 天
    deleted, _ := c.CleanupHistory(time.Now().Add(-30 * 24 * time.Hour))
    log.Printf("清理了 %d 条记录", deleted)
})
```

### Q: 可以禁用历史记录吗？

A: 可以。不配置 `WithHistoryRecorder` 选项即可：

```go
// 不启用历史记录
c := cron.New()  // 不传递 WithHistoryRecorder
```

### Q: 历史记录文件损坏怎么办？

A: JSONL 格式的优势是单条记录损坏不影响其他记录。只需删除损坏的行即可。

### Q: 如何迁移到数据库存储？

## Cron 库性能优化

### 1. 任务配置优化

#### 合理使用异步执行

```go
// 对于耗时任务，使用异步执行避免阻塞调度器
c.Schedule("heavy-task", "*/5 * * * * *", func(ctx context.Context) {
    // 耗时操作
    processLargeDataset()
}, cron.JobOptions{
    Async: true,  // 异步执行，不阻塞调度循环
})

// 对于轻量级任务，使用同步执行减少 goroutine 开销
c.Schedule("light-task", "*/1 * * * * *", func(ctx context.Context) {
    // 快速操作
    updateCache()
}, cron.JobOptions{
    Async: false,  // 同步执行，减少资源消耗
})
```

#### 设置合理的超时时间

```go
// 设置超时避免任务长时间运行
c.Schedule("api-call", "*/10 * * * * *", func(ctx context.Context) {
    // 任务会在 30 秒后自动超时
    callExternalAPI(ctx)
}, cron.JobOptions{
    Timeout: 30 * time.Second,
})
```

#### 并发数控制

```go
// 限制并发数避免资源耗尽
c.Schedule("db-sync", "*/5 * * * * *", func(ctx context.Context) {
    syncDatabase()
}, cron.JobOptions{
    MaxConcurrent: 3,  // 最多同时运行 3 个实例
})
```

### 2. 调度表达式优化

#### 避免过于频繁的调度

```go
// ❌ 避免：每秒执行可能导致性能问题
c.Schedule("check", "* * * * * *", handler)

// ✅ 推荐：根据实际需求选择合适的间隔
c.Schedule("check", "*/5 * * * * *", handler)  // 每 5 秒
c.Schedule("check", "0 * * * * *", handler)    // 每分钟
```

#### 使用 @every 语法

```go
// @every 语法更高效，避免重复解析
c.Schedule("periodic", "@every 10s", handler)
c.Schedule("hourly", "@every 1h", handler)
```

#### 分散任务执行时间

```go
// ❌ 避免：所有任务同时执行造成峰值
c.Schedule("task1", "0 * * * * *", handler1)  // 每分钟整点
c.Schedule("task2", "0 * * * * *", handler2)  // 每分钟整点
c.Schedule("task3", "0 * * * * *", handler3)  // 每分钟整点

// ✅ 推荐：错开执行时间，平滑负载
c.Schedule("task1", "0 * * * * *", handler1)   // 0 秒执行
c.Schedule("task2", "20 * * * * *", handler2)  // 20 秒执行
c.Schedule("task3", "40 * * * * *", handler3)  // 40 秒执行
```

### 3. 任务实现优化

#### 提前退出检查

```go
// 在长时间运行的任务中检查上下文取消信号
c.Schedule("long-task", "@every 1m", func(ctx context.Context) {
    for i := 0; i < 10000; i++ {
        // 每次迭代都检查是否需要退出
        select {
        case <-ctx.Done():
            return  // 优雅退出
        default:
            processItem(i)
        }
    }
})
```

#### 批量处理

```go
// 批量处理数据，减少系统调用
c.Schedule("batch-process", "@every 5m", func(ctx context.Context) {
    const batchSize = 100

    items := fetchItems()
    for i := 0; i < len(items); i += batchSize {
        end := i + batchSize
        if end > len(items) {
            end = len(items)
        }

        // 批量处理
        processBatch(items[i:end])
    }
})
```

#### 连接复用

```go
// 复用数据库连接和 HTTP 客户端
var (
    db         *sql.DB
    httpClient *http.Client
)

func init() {
    db = initDatabase()
    httpClient = &http.Client{
        Timeout: 10 * time.Second,
        Transport: &http.Transport{
            MaxIdleConns:        100,
            MaxIdleConnsPerHost: 10,
        },
    }
}

c.Schedule("db-task", "@every 1m", func(ctx context.Context) {
    // 复用连接池，避免频繁建立连接
    rows, _ := db.QueryContext(ctx, "SELECT * FROM tasks")
    defer rows.Close()
    // 处理数据
})
```

### 4. 内存优化

#### 避免内存泄漏

```go
// ✅ 正确：确保资源释放
c.Schedule("resource-task", "@every 30s", func(ctx context.Context) {
    file, err := os.Open("data.txt")
    if err != nil {
        return
    }
    defer file.Close()  // 确保文件关闭

    // 处理文件
})

// ❌ 错误：可能导致文件句柄泄漏
c.Schedule("leak-task", "@every 30s", func(ctx context.Context) {
    file, _ := os.Open("data.txt")
    // 缺少 defer file.Close()
})
```

#### 控制内存分配

```go
// 预分配切片容量，避免频繁扩容
c.Schedule("slice-task", "@every 1m", func(ctx context.Context) {
    items := make([]Item, 0, 1000)  // 预分配容量

    for i := 0; i < 1000; i++ {
        items = append(items, Item{ID: i})
    }

    processItems(items)
})
```

### 5. 历史记录优化

#### 定期清理历史记录

```go
// 定期清理旧记录，控制存储空间
c.Schedule("cleanup-history", "@daily", func(ctx context.Context) {
    thirtyDaysAgo := time.Now().Add(-30 * 24 * time.Hour)
    deleted, err := c.CleanupHistory(thirtyDaysAgo)
    if err != nil {
        log.Printf("清理历史记录失败: %v", err)
        return
    }
    log.Printf("已清理 %d 条历史记录", deleted)
})
```

#### 按需启用历史记录

```go
// 仅对关键任务启用历史记录
storage, _ := history.NewFileStorage("./history")
recorder := history.NewHistoryRecorder(storage)
c := cron.New(cron.WithHistoryRecorder(recorder))

// 关键任务：启用历史记录（通过调度器配置）
c.Schedule("critical-task", "@every 1m", criticalHandler)

// 非关键任务：不影响历史记录（所有任务共享调度器的历史配置）
c.Schedule("optional-task", "@every 5m", optionalHandler)
```

## Go 运行时优化

### 1. 垃圾回收调优

```bash
# 设置 GC 目标百分比（默认 100）
export GOGC=50  # 更频繁的 GC，减少内存使用

# 设置内存限制（Go 1.19+）
export GOMEMLIMIT=2GiB

# 启用 GC 跟踪
export GODEBUG=gctrace=1
```

### 2. 调度器调优

```bash
# 设置最大处理器数（默认为 CPU 核心数）
export GOMAXPROCS=8

# 启用调度器跟踪
export GODEBUG=schedtrace=1000
```

### 3. 应用示例

```go
func main() {
    // 在代码中设置 GOMAXPROCS
    runtime.GOMAXPROCS(8)

    // 设置 GC 百分比
    debug.SetGCPercent(50)

    // 创建调度器
    c := cron.New()
    // 添加任务...
    c.Start()
}
```

## 性能监控

### 1. 内置监控指标

```go
// 定期检查任务统计
go func() {
    ticker := time.NewTicker(time.Minute)
    defer ticker.Stop()

    for range ticker.C {
        stats := c.GetAllStats()
        for id, stat := range stats {
            // 输出性能指标
            log.Printf("任务 %s: 运行 %d 次, 成功 %d 次, 失败 %d 次, 平均耗时 %v",
                id,
                stat.RunCount,
                stat.SuccessCount,
                stat.FailCount,
                stat.AvgDuration,
            )

            // 检查性能问题
            if stat.AvgDuration > time.Minute {
                log.Printf("警告: 任务 %s 平均耗时过长", id)
            }

            successRate := float64(stat.SuccessCount) / float64(stat.RunCount) * 100
            if successRate < 95 {
                log.Printf("警告: 任务 %s 成功率过低: %.2f%%", id, successRate)
            }
        }
    }
}()
```

### 3. 事件钩子监控

```go
// 使用事件钩子监控任务执行
hook := func(ev cron.Event) {
    if ev.End.IsZero() {
        return  // 跳过开始事件
    }

    // 记录执行时间
    log.Printf("任务 %s 执行完成: 成功=%v, 耗时=%v, 重试=%d",
        ev.TaskID, ev.Success, ev.Duration, ev.Retries)

    // 监控慢任务
    if ev.Duration > 30*time.Second {
        log.Printf("慢任务警告: %s 耗时 %v", ev.TaskID, ev.Duration)
    }

    // 监控失败任务
    if !ev.Success {
        log.Printf("任务失败: %s, 错误: %s", ev.TaskID, ev.Error)
    }
}

c := cron.New(cron.WithEventHook(hook))
```

## 性能问题排查

### 1. 任务执行慢

**诊断步骤：**

```go
// 检查任务统计
stats, ok := c.GetStats("slow-task")
if ok {
    fmt.Printf("平均耗时: %v\n", stats.AvgDuration)
    fmt.Printf("最长耗时: %v\n", stats.MaxDuration)
}
```

**可能原因：**
- 任务实现效率低
- 外部依赖响应慢
- 资源竞争
- 并发数设置不合理

**解决方案：**
- 优化任务代码
- 设置超时时间
- 使用异步执行
- 增加并发数或使用批处理

### 2. 内存占用高

**诊断步骤：**

```bash
# 生成内存分析文件
curl http://localhost:6060/debug/pprof/heap > heap.prof

# 分析内存使用
go tool pprof heap.prof
```

**可能原因：**
- 任务创建过多对象
- 历史记录未清理
- 存在内存泄漏
- GC 不够频繁

**解决方案：**
- 减少对象分配
- 定期清理历史记录
- 检查资源释放
- 调整 GOGC 参数

### 3. CPU 占用高

**诊断步骤：**

```bash
# 生成 CPU 分析文件
curl http://localhost:6060/debug/pprof/profile > cpu.prof

# 分析 CPU 使用
go tool pprof cpu.prof
```

**可能原因：**
- 任务调度过于频繁
- 任务计算密集
- 并发数过高
- 上下文切换频繁

**解决方案：**
- 降低调度频率
- 优化计算逻辑
- 调整并发数
- 增加 GOMAXPROCS

## 最佳实践总结

1. **任务设计原则**
   - 任务应该快速完成（建议 < 1 秒）
   - 耗时任务使用异步执行
   - 实现优雅退出机制
   - 合理设置超时时间

2. **资源管理原则**
   - 复用数据库连接和 HTTP 客户端
   - 及时释放文件句柄和其他资源
   - 使用对象池减少 GC 压力
   - 预分配切片容量

3. **监控告警原则**
   - 监控关键性能指标
   - 设置合理的告警阈值
   - 定期分析性能数据
   - 及时响应性能问题

4. **容量规划原则**
   - 根据实际负载调整配置
   - 预留足够的性能余量
   - 定期进行压力测试
   - 制定扩容方案
